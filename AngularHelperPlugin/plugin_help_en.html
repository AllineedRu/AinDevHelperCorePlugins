Angular Helper Plugin - Documentation

About the plugin

A plugin that provides Angular developers with various useful actions, such as obtaining information about Angular CLI, NPM, Node.js versions, and also makes it easier to create new applications and components in Angular.

Supported plugin actions

The plugin supports the following actions:

  • Show currently active version of Angular CLI
  • Show the current active version of Node.js
  • Show currently active NPM version
  • Create a new Angular application via Angular CLI
  • Create a new Angular component via Angular CLI
  • Create a new Angular service via Angular CLI
  • Create a new configuration file via Angular CLI
  • Create a new library via Angular CLI
  • Create a new module via Angular CLI
  • Create a new enum via Angular CLI
  • Create a new interface via Angular CLI
  • Create a new resolver via Angular CLI
  • Create a new pipe via Angular CLI
  • Create a new directive via Angular CLI
  • Create service-worker via Angular CLI
  • Create a new web-worker via Angular CLI
  • Create a new guard via Angular CLI
  • Create a new interceptor via Angular CLI
  • Generate environments for a project via Angular CLI

Description of actions

Below is a description of how the actions supported by the plugin work.

Show currently active version of Angular CLI

The action displays the currently active version of Angular CLI installed on the machine.

The plugin executes the command cmd /c ng version 1> angular_cli_version.txt, starting the process cmd with arguments ng version 1> angular_cli_version.txt and the /c key, so the result of the command is written to the temporary file angular_cli_version.txt.

Show the current active version of Node.js

The action displays the current active version of Node.js installed on the machine.

When the action is executed, the plugin executes the command cmd /c node -v 1> node_version.txt, starting the cmd process with the arguments node -v 1> node_version.txt and the key /c, so the result of the command is written to the temporary file node_version.txt.

Show currently active NPM version

Create a new Angular application via Angular CLI

A parameterized action allows you to create a new Angular application by executing the appropriate command through the Angular CLI.

When the action is executed, the plugin executes the command cmd /c ng new ${appName} 1> new_angular_app.txt, starting the cmd process with the arguments ng new ${appName} 1> new_angular_app.txt and the key /c, so the result of the command is written to a temporary file new_angular_app.txt.

Action Parameters

  • appName - name of the new Angular application
  • appPath - working directory for creating a new Angular application when creating it and starting the cmd process

Create a new Angular component via Angular CLI

A parameterized action allows you to create a new component in an already created Angular application using the Angular CLI tools.

When the action is executed, the plugin executes the command cmd /c ng generate component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt, starting the cmd process with arguments ng generate component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt and key /c, thus the result of the command execution is written to the temporary file new_angular_component.txt if there are no command execution errors and to the file new_angular_component_error.txt if there are errors.

Action Parameters

  • appPath - a directory with an already created Angular application in which you want to create a new component
  • newComponentName - name of the new Angular component
