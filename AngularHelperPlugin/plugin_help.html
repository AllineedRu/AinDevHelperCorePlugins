Angular Helper Plugin - Документация

О плагине

Плагин, предоставляющий разработчикам на Angular различные полезные действия, такие как получение информации о версиях Angular CLI, NPM, Node.js, а также облегчающий создание новых приложений и компонентов на Angular.

Поддерживаемые действия плагина

Плагин поддерживает следующие действия:

  • Показать текущую активную версию Angular CLI
  • Показать текущую активную версию Node.js
  • Показать текущую активную версию NPM
  • Создать новое Angular приложение через Angular CLI
  • Создать новый Angular компонент через Angular CLI
  • Создать новый Angular сервис через Angular CLI
  • Создать новый конфигурационный файл через Angular CLI
  • Создать новую библиотеку через Angular CLI
  • Создать новый модуль через Angular CLI
  • Создать новый enum через Angular CLI
  • Создать новый интерфейс через Angular CLI
  • Создать новый резолвер (resolver) через Angular CLI
  • Создать новый пайп (pipe) через Angular CLI
  • Создать новую директиву (directive) через Angular CLI
  • Создать service-worker через Angular CLI
  • Создать web-worker через Angular CLI
  • Создать защитника (guard) через Angular CLI
  • Создать перехватчика (interceptor) через Angular CLI
  • Сгенерировать окружения (environments) для проекта через Angular CLI

Описание действий

Ниже приводится описание работы действий, поддерживаемых плагином.

Показать текущую активную версию Angular CLI

Действие отображает текущую активную версию Angular CLI, установленную на компьютере.

Плагин выполняет команду cmd /c ng version 1> angular_cli_version.txt, запуская процесс cmd с аргументами ng version 1> angular_cli_version.txt и ключом /c, таким образом результат выполнения команды записывается во временный файл angular_cli_version.txt.

Создать новое Angular приложение через Angular CLI

Параметризованное действие позволяет создать новое приложение Angular при помощи выполнения соответствующей команды через Angular CLI.

При выполнении действия плагин выполняет команду cmd /c ng new ${appName} 1> new_angular_app.txt, запуская процесс cmd с аргументами ng new ${appName} 1> new_angular_app.txt и ключом /c, таким образом результат выполнения команды записывается во временный файл new_angular_app.txt.

Параметры действия

  • appName - название нового Angular приложения
  • appPath - рабочий каталог для создания нового Angular приложения при его создании и запуске процесса cmd

Параметризованное действие позволяет создать через средства Angular CLI новый компонент в уже созданном приложении Angular.

При выполнении действия плагин выполняет команду cmd /c ng generate component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt, запуская процесс cmd с аргументами ng generate component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt и ключом /c, таким образом, результат выполнения команды записывается во временный файл new_angular_component.txt при отсутствии ошибок исполнения команды и в файл new_angular_component_error.txt при наличии ошибок.

Параметры действия

  • appPath - каталог с уже созданным приложением Angular, в котором требуется создать новый компонент
