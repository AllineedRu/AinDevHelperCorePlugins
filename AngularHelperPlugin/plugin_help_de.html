Angular Helper Plugin – Dokumentation

Über das Plugin

Ein Plugin, das Angular-Entwicklern verschiedene nützliche Aktionen bietet, z. B. das Abrufen von Informationen zu Angular-CLI-, NPM- und Node.js-Versionen, und das außerdem die Erstellung neuer Anwendungen und Komponenten in Angular erleichtert.

Unterstützte Plugin-Aktionen

Das Plugin unterstützt die folgenden Aktionen:

  • Zeigt die aktuell aktive Version von Angular CLI an
  • Zeigt die aktuell aktive Version von Node.js an
  • Zeigt die aktuell aktive NPM-Version an
  • Erstellen Sie eine neue Angular-Anwendung über die Angular-CLI
  • Erstellen Sie eine neue Angular-Komponente über die Angular-CLI

Beschreibung der Aktionen

Nachfolgend finden Sie eine Beschreibung, wie die vom Plugin unterstützten Aktionen funktionieren.

Zeigt die aktuell aktive Version von Angular CLI an

Die Aktion zeigt die derzeit aktive Version von Angular CLI an, die auf dem Computer installiert ist.

Das Plugin führt den Befehl cmd /c ng version 1> angle_cli_version.txt aus und startet den Prozess cmd mit den Argumenten ng version 1> angle_cli_version.txt und den Schlüssel /c, damit das Ergebnis des Befehls in die temporäre Datei angular_cli_version geschrieben wird. txt.

Zeigt die aktuell aktive Version von Node.js an

Die Aktion zeigt die aktuell aktive Version von Node.js an, die auf dem Computer installiert ist.

Wenn die Aktion ausgeführt wird, führt das Plugin den Befehl cmd /c node -v 1> node_version.txt aus und startet den cmd Prozess mit den Argumenten node -v 1> node_version.txt und dem Schlüssel /c, sodass das Ergebnis des Befehls geschrieben wird in die temporäre Datei node_version.txt.

Zeigt die aktuell aktive NPM-Version an

Die Aktion zeigt die aktuell aktive Version von NPM an, die auf dem Computer installiert ist.

Wenn die Aktion ausgeführt wird, führt das Plugin den Befehl cmd /c npm -v 1> npm_version.txt aus und startet den Prozess cmd mit den Argumenten node -v 1> node_version.txt und dem Schlüssel /c, damit das Ergebnis des Befehls geschrieben wird in die temporäre Datei npm_version.txt.

Erstellen Sie eine neue Angular-Anwendung über die Angular-CLI

Mit einer parametrisierten Aktion können Sie eine neue Angular-Anwendung erstellen, indem Sie den entsprechenden Befehl über die Angular-CLI ausführen.

Wenn die Aktion ausgeführt wird, führt das Plugin den Befehl cmd /c ng new ${appName} 1> new_angular_app.txt aus und startet den cmd-Prozess mit den Argumenten ng new ${appName} 1> new_angular_app.txt und dem Schlüssel /c, also Das Ergebnis des Befehls wird in eine temporäre Datei new_angular_app.txt geschrieben.

Aktionsparameter

  • appName - Name der neuen Angular-Anwendung
  • appPath - Arbeitsverzeichnis zum Erstellen einer neuen Angular-Anwendung beim Erstellen und Starten des cmd-Prozesses

Erstellen Sie eine neue Angular-Komponente über die Angular-CLI

Mit einer parametrisierten Aktion können Sie mithilfe der Angular-CLI-Tools eine neue Komponente in einer bereits erstellten Angular-Anwendung erstellen.

Wenn die Aktion ausgeführt wird, führt das Plugin den Befehl cmd /c ng generic Component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt aus und startet damit die cmd-Prozess mit Argumenten ng generic Component ${newComponentName} 1> new_angular_component.txt 2> new_angular_component_error.txt und Schlüssel /c, somit wird das Ergebnis der Befehlsausführung in die temporäre Datei new_angular_component.txt geschrieben, wenn keine Befehlsausführungsfehler vorliegen, und in die Datei new_angular_component_error wenn es Fehler gibt.

Aktionsparameter

  • appPath - ein Verzeichnis mit einer bereits erstellten Angular-Anwendung, in der Sie eine neue Komponente erstellen möchten
  • newComponentName - Name der neuen Angular-Komponente
